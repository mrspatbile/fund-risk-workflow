# Runbook

## Environment setup

#### 1. Create and activate virtual environment
```bash 
python3.13 -m venv .venv
source .venv/bin/activate
```

#### 2. Install all dependencies (editable so src/ imports work)
```bash 
pip install -e .
```

#### 3. Install Playwright browser (required for save_table_png)
```bash 
playwright install chromium
```

#### 4. Database initialisation and synthetic data generation

Step 1 — rebuild schema, regenerate all fund data, reload and enrich positions
```bash 
python -m src.setup_db --force
```

Step 2 — generate daily fund-admin export slices (fake FA files per fund/date)
```bash 
python -m src.data.generate_daily_export
```

What `--force` does, in order:

- Drops and recreates the full schema
- Regenerates 250-day Excel position histories for the funds (HF, PD, RE, UCITS)
- Loads positions into positions table
- Enriches positions → positions_enriched (Bloomberg sensitivities, fund-admin data)
- Runs `generate_pe_fund` → PE tables
- Runs `generate_infra_fund` → infra tables (assets, NAV history, debt, covenants)
- The daily export is not part of `setup_db` — always run it separately after.

#### 5. Regenerating individual fund data
If you only need to regenerate one component without a full DB rebuild:


5.1. Liquid fund Excel files only (HF, PD, RE, UCITS)
```bash 
python -m src.data.generate_positions
```

5.2.  PE fund tables only (requires DB schema already exists)
```bash 
python -c "from src.data.database import get_engine; from src.data.generate_pe_fund import generate_pe_fund; generate_pe_fund(get_engine())"
```

5.3.  Infra fund tables only (requires DB schema already exists)
```bash 
python -c "from src.data.database import get_engine; from src.data.generate_infra_fund import generate_infra_fund; generate_infra_fund(get_engine())"
```

5.4. Validate the full pipeline
```bash
python -m src.validate_pipeline
```


## Notebook execution sequence
Open the notebook from **project root** with the `venv` active.

Notebooks are in the folder `notebooks/`. Recommended execution order:

**validation**

1. `data_pipeline.ipynb`: runs pricing validation.

**individual funds examples: UCITS and AIFMs**

2. `ucits_balanced.ipynb`: includes VaR limits, SRRI, eligibility, pre-trade
3. `aifm_hedge_fund.ipynb`: includes leverage, stress, liquidity, pre-trade, Annex IV
4. `aifm_private_debt.ipynb`: includes credit risk, leverage, ESG, Annex IV
5. `aifm_real_estate.ipynb`: includes LTV, rental stress, direct property, Annex IV
6. `aifm_pe_buyout.ipynb`: includes IRR, multiples, PME (public markets equivalent), ESG, Annex IV
7. `aifm_infra_fund.ipynb`: includes DSCR/LTV, duration, stress, ESG, Annex IV

**Dashboard**

8. `board_risk_report.ipynb`: Drives board_report.py → PDF.

> Note 1: All notebooks use `VALUATION_DATE = '2026-05-13'` (static).

> Note 2: All charts and tables saved during notebook execution are written to `reports/<fund_id>/` as PNG files via `save_fig` and `save_table_png`. These are standalone artifacts for reference and sharing. 

## Annex IV export (AIFMD Art. 110)
Covers all AIFM funds in teh database. Run from a notebook or script:

```python
from src.data.database import get_engine
from src.annex_iv import export_annex_iv_excel

ENGINE = get_engine()
quarter = '26Q1'

export_annex_iv_excel(ENGINE, quarter=quarter)

```
Output: `data/annex_iv_report_<quarter>.xlsx`

## Board Risk Report PDF (AIFMD Art. 15)

The report is generated by calling `generate_board_report()` from `board_report.py`.

The notebook `board_risk_report.ipynb`is the standard entry point. As a shortcut, run directly:

```bash
python -m src.board_report
```
Output: `data/board_risk_report_<date>.pdf`

The board report produces a single self-contained PDF covering the full AIFM. It is a cross-fund view designed for board and risk committee consumption.


## Common errors and fixes

#### Module not found
```bash
ModuleNotFoundError: src
```
Run from the project root with the `venv` active. Never run from inside `src/`.

---

#### Table not found

```bash
no such table: positions
DB schema missing. 
```

Build the database by running:
```bash 
python -m src.setup_db --force
```
---

#### `yfinance` returns empty prices

Price cache may be stale or the ticker has changed. Delete the relevant file in
`data/yf_cache/` and rerun `python -m src.setup_db`. The data from `yfinance` will refetch automatically.

---

#### Playwright TimeoutError on `save_table_png`

Run `playwright install chromium` to ensure the browser binary is present.

---


#### nest_asyncio error in notebook

`nest_asyncio.apply()` is called at import time in `nb_utils.py`. If it errors,
ensure `nest_asyncio` is installed: `pip install nest_asyncio`.

---


#### Infra covenant breach count unexpected
`generate_infra_fund` uses a fixed random seed per asset. If you change noise
profiles or capital structure in `generate_infra_fund.py`, rerun
`python -m src.setup_db --force` to regenerate. 

The Example baked in the infra portfolio has "intentionally" 2 designed breaches: 
- INFRA_003 Q2 2020 DSCR
- INFRA_007 Q3 2023 LTV
